# StreamSuites-Creator v0.4.2-alpha

`StreamSuites-Creator v0.4.2-alpha` focuses on creator-surface correctness and release hardening rather than a brand-new product area. The cycle centers on auth-gate alignment with the runtime, cleaner creator account and routing behavior, and polish that keeps the dashboard stable while the platform-wide access model evolves.

## 🔐 Auth and Session Cleanup

- Wired creator auth entry points to the runtime-owned `/auth/access-state` and `/auth/debug/unlock` contract instead of letting the frontend invent its own maintenance behavior.
- Repaired creator login redirect loops, callback timing, post-login hydration, and auth-session noise so valid creator sessions settle cleanly.
- Preserved creator-specific soft lockout behavior for invalid-session and role-mismatch states without moving backend ownership into this repo.

## 🧭 Account, Routing, and Profile Flow

- Moved the creator surface further toward clean path routing and Cloudflare Pages-safe deep links.
- Added creator-facing public profile and FindMeHere settings, including canonical slug and visibility controls backed by the authoritative profile APIs.
- Tightened profile settings load and save behavior so the account surface remains truthful to the upstream public-profile model.

## 🎨 UX Stabilization and Surface Polish

- Refined auth notices, modal styling, footer version/build presentation, and shell layout so maintenance messaging and creator dashboard affordances feel consistent with the broader platform.
- Continued creator dashboard polish across settings, notifications, overview, statistics, and Discord integration surfaces.
- Kept scoped profile previews and badge rendering aligned with the shared public and admin hovercard model.
